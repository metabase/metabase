<!-- The preset system pattern from src/metabase/logger/api.clj -->

The `presets` function returns a vector of preset maps. Each preset has this shape:

```clojure
{:id :sync
 :display_name (tru "Sync issue troubleshooting")
 :loggers (doto (->> (concat (loggers-under "metabase.sync")
                             (loggers-under "metabase.driver.sql-jdbc.sync"))
                     (filter map?)
                     (map #(assoc % :level :debug))
                     (sort-by :name)
                     (vec))
              (as-> $ (when (empty? $) (log/error "Sync preset is empty"))))}
```

Key helpers already available:
- `(loggers-under "namespace.prefix")` - returns logger maps for all loaded namespaces under a prefix
- `(tru "...")` - i18n translation macro
- Each logger map: `{:name "namespace.name" :level :debug}`

The presets are served via `GET /api/logger/presets` and applied via `POST /api/logger/adjustment` with the loggers' namespace->level pairs.

From Phase 2, the namespace that emits per-statement DEBUG logs is:
- `metabase.driver.sql-jdbc.execute` (contains `emit-statement-detail!`)

From Phase 1, the namespace that emits INFO summary logs is:
- `metabase.query-processor.middleware.query-log` (always-on at INFO, but its DEBUG level is also useful)
