# i18n.spec.ts (from i18n/i18n.cy.spec.ts)

10 tests (one per locale), 8 pages each. Ported faithfully. Green on the jar
(slot 3), 40/40 across two consecutive `--repeat-each=2` runs. tsc clean.

## Fixes / classification

- **Gotcha (harness, not app): `page.goto("/")` flakily throws
  `net::ERR_ABORTED`.** The homepage fires a client-side redirect that
  supersedes the in-flight navigation before it commits, so goto rejects even
  with `waitUntil: "commit"` — while the target page renders fine (the failure
  snapshot showed the fully-rendered, translated UI). Cypress's `cy.visit`
  tolerated this internally. Ported via `visitPath` (support/i18n.ts): swallow
  the specific `ERR_ABORTED`, then `waitForLoadState("domcontentloaded")`; the
  caller's retrying `getByRole("main")` / `error-boundary` assertions validate
  the final page. Reproduced only under `--repeat-each` load (1/20 then 1/20),
  never in the plain run — the transient-navigation analogue of the
  transient-UI strict-mode class. No cross-check needed: rendered page proves
  it's a navigation-timing artifact, not a product bug.
